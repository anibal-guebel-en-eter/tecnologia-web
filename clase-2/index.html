¿Qué es Gutenberg?

El Editor de Bloques

🧩Concepto de Bloques

Cada elemento (texto, imagen, botón) es un bloque independiente y reutilizable

🎨WYSIWYG Verdadero

What You See Is What You Get - El frontend se ve exactamente como en el editor

🔧Sin Código

Diseño visual profesional sin tocar HTML/CSS

📱Responsive Nativo

Adaptación automática a móviles y tablets

¿Qué son los Patrones?

Block Patterns Revolution

✨Definición:

Combinaciones prediseñadas de bloques que crean secciones completas (hero, testimonios, precios, etc.)

⚡Velocidad

Página profesional en minutos, no horas

🎯Consistencia

Diseño coherente en todo el sitio

🔄Reutilización

Un patrón → múltiples páginas

✏️Personalización

Base sólida, ajustes específicos

Reutilizar Bloques

Eficiencia Máxima

Crear bloque perfectamente diseñado

Keyboard Shortcuts

Productividad Pro

⌨️ Navegación

  • Tab: siguiente bloque
  • Shift+Tab: bloque anterior
  • Ctrl+A: seleccionar todo

✏️ Edición

  • Ctrl+Z: deshacer
  • Ctrl+Y: rehacer
  • Ctrl+C/V: copiar/pegar

🚀 Rápidos

  • Ctrl+S: guardar
  • Ctrl+Shift+Y: list view
  • /: insertar bloque

🎨 Formato

  • Ctrl+B: negrita
  • Ctrl+I: cursiva
  • Ctrl+K: enlace

Gutenberg vs Page Builders

¿Por qué elegir el editor nativo?

Page Builders (Elementor, Divi) Gutenberg Nativo
💰 Costo adicional 🆓 Incluido en WordPress
🐌 Código adicional = lentitud ⚡ Optimizado, rápido
🔒 Dependencia del plugin 🔓 Estándar de WordPress
📚 Curva de aprendizaje específica 🎯 Skills transferibles
🔄 Problemas de migración ✅ Contenido portable

Troubleshooting Común

Problemas y Soluciones

❌

Problema: "Bloque no reconocido" después de cambiar tema

Solución: Convertir a HTML o buscar bloque equivalente

🐌

Problema: Editor lento con muchos bloques

Solución: Usar bloques reutilizables y patrones

📱

Problema: Diseño roto en móvil

Solución: Revisar configuraciones responsive por bloque

🎨

Problema: Colores no coinciden con la marca

Solución: Personalizar paleta global en el Site Editor
